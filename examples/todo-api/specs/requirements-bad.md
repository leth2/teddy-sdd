# 요구사항: todo-api

## 개요
TODO를 관리하는 REST API를 만든다. 사용자가 할 일을 적절히 관리할 수 있어야 한다.

## 요구사항

### 1. 인증

**1.1** 사용자는 로그인할 수 있어야 한다.

**1.2** 토큰은 적절한 시간 동안 유효해야 한다.

**1.3** 인증이 필요한 API는 토큰을 확인해야 한다.

### 2. TODO 관리

**2.1** 사용자는 TODO를 생성할 수 있어야 한다.
- 제목은 필요에 따라 입력한다.
- 내용은 상황에 맞게 처리한다.

**2.2** TODO 목록을 조회할 수 있어야 한다.
- 필요한 경우 필터링이 가능해야 한다.

**2.3** TODO를 수정할 수 있어야 한다.

**2.4** TODO를 삭제할 수 있어야 한다.

### 3. 태그

**3.1** TODO에 태그를 붙일 수 있어야 한다.

**3.2** 태그로 TODO를 분류할 수 있어야 한다.

## 제약 조건

- 성능은 적절히 보장되어야 한다.
- 보안은 일반적인 수준으로 유지한다.
