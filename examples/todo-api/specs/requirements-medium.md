# 요구사항: todo-api (중간 품질)

## 개요
인증된 사용자가 TODO 항목을 생성·조회·수정·삭제할 수 있는 REST API.

## 요구사항

### 1. 인증

**REQ-001** 사용자는 이메일과 비밀번호로 로그인할 수 있어야 한다.
- 로그인 성공 시 JWT 토큰을 반환한다.
- 잘못된 자격증명의 경우 오류를 반환해야 한다.
- 토큰은 7일간 유효하다.

**REQ-002** 인증이 필요한 API에 토큰 없이 요청하면 401을 반환한다.
- 만료된 토큰도 401을 반환해야 한다.

**REQ-003** 비밀번호는 bcrypt로 해싱하여 저장한다.

### 2. TODO 관리

**REQ-004** 인증된 사용자는 TODO를 생성할 수 있다.
- 제목(title)은 필수, 최대 200자이다.
- 설명(description)은 선택사항이다.
- 생성 시 HTTP 201을 반환한다.
- 제목이 없으면 400을 반환한다.

**REQ-005** 인증된 사용자는 자신의 TODO 목록을 조회할 수 있다.
- 페이지네이션을 지원한다 (기본: page=1, limit=20).
- 다른 사용자의 TODO는 반환하지 않는다.

**REQ-006** 인증된 사용자는 자신의 TODO를 수정할 수 있다.
- 존재하지 않는 TODO는 404를 반환한다.
- 다른 사용자의 TODO 수정은 허용하지 않는다.

**REQ-007** 인증된 사용자는 자신의 TODO를 삭제할 수 있다.
- 삭제 성공 시 204를 반환한다.

### 3. 태그

**REQ-008** TODO에 태그를 붙일 수 있다.
- 태그는 최대 10개까지 허용한다.

**REQ-009** 태그로 TODO를 필터링할 수 있다.
- 존재하지 않는 태그로 필터링하면 빈 목록을 반환한다.

## 제약 조건

- SQL Injection 방지를 위해 parameterized query를 사용한다.
